# vibestart — Инструкция на русском

**От нуля до вайбкодинга за один промпт.**

Ты только что установил VS Code. Хочешь что-то построить с помощью ИИ.  
Никогда не открывал терминал. Не знаешь что такое npm или git.  
Всё нормально.

---

## Как это работает

**Шаг 1.** Установи Kilo Code в VS Code:  
→ [Kilo Code в Marketplace](https://marketplace.visualstudio.com/items?itemName=kilocode.Kilo-Code)  
(VS Code → значок Extensions слева → найди "Kilo Code" → Install)

**Шаг 2.** Открой чат Kilo Code  
(`Ctrl+Shift+P` → напечатай "Kilo Code: Open Chat" → Enter)

**Шаг 3.** Открой файл [PROMPT.md](../PROMPT.md), скопируй всё после первой строки, вставь в чат

**Шаг 4.** Отвечай на вопросы агента о своём проекте

**Шаг 5.** Напечатай: `Initialize according to custom instructions.`

Готово. Агент всё настроит и будет помнить твой проект между сессиями.

---

## Что агент делает за тебя

| Что | Зачем |
|-----|-------|
| git, Node.js, uv | Инструменты, нужные для работы ИИ |
| GRACE | Методология — держит ИИ сфокусированным на твоём проекте |
| ConPort | Память — ИИ помнит твой проект между сессиями |
| `AGENTS.md` | Инструкция для ИИ по работе с твоим проектом |
| `projectBrief.md` | Описание проекта (заполняется вместе с тобой) |

---

## Если что-то не работает

Агент сам разберётся с большинством проблем.  
Если он просит что-то сделать вручную — он объяснит почему и даст точную инструкцию.

Если возникли вопросы: открой [issue на GitHub](https://github.com/xronocode/vibestart/issues).

---

## Как начинать каждую новую сессию

Каждый раз когда открываешь проект заново, напечатай в чат Kilo Code:

```
Initialize according to custom instructions.
```

Агент загрузит контекст и вспомнит где вы остановились.

---

## Почему GRACE + ConPort вместе?

Подробнее: [docs/why.md](why.md)

Коротко: оба инструмента хранят информацию о проекте, но для разных целей.  
Без чётких правил агент начинает дублировать данные и через пару дней  
никто не знает что актуально. vibestart решает это с первого дня.
