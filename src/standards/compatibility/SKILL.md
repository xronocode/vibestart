# Skill Compatibility Standard

## Purpose

Detect and resolve conflicts between:
- Existing agent skills (grace-marketplace, custom)
- Configuration files (AGENTS.md, .cursorrules, etc.)
- GRACE artifacts (docs/*.xml)
- Session logs and project files

## Conflict Types

### CONF-001: Duplicate Skills

**Description:** Same skill from different sources

**Example:** grace-init from vibestart vs grace-marketplace

**Detection:**
```
Locations:
  - ~/.kilocode/skills/
  - .kilocode/skills/
  - .cursor/skills/
  - .claude/skills/

Marker: SKILL.md with same "name:" field
```

**Resolution Options:**
1. **vibestart-override** (recommended)
   - Backup originals to ~/.vibestart/backups/
   - Install vibestart v2.0 versions
   - Update skills-registry.json
   
2. **keep-original**
   - Keep existing skills unchanged
   - Only add vs-* skills
   - No GRACE enhancements
   
3. **merge-manual**
   - Review each conflict manually
   - Choose per-skill

---

### CONF-002: AGENTS.md Conflict

**Description:** Existing AGENTS.md vs generated from vibestart

**Detection:**
```
Check file: AGENTS.md

Markers:
  - "Generated from vibestart" → Already managed
  - "GRACE Framework" → grace-marketplace or manual
  - No markers → Unknown source
```

**Scenarios:**

| Scenario | Condition | Meaning | Risk |
|----------|-----------|---------|------|
| S1 | Exists, no generation marker | Manual or another framework | High - may lose custom rules |
| S2 | Exists, old vibestart marker | Generated by older version | Medium - may need refresh |
| S3 | Exists, current vibestart marker | Already managed | Low - just update |

**Resolution Options:**
1. **backup-and-regenerate**
   - Save original to AGENTS.md.backup
   - Generate new from vibestart fragments
   
2. **keep-add-marker**
   - No content changes
   - Add generation metadata
   
3. **merge-preserve-custom**
   - Extract custom sections
   - Generate base from fragments
   - Merge custom back
   
4. **keep-original**
   - No changes

---

### CONF-003: Configuration Conflict

**Description:** Multiple configuration files with overlapping settings

**Detection:**
```
Files to check:
  - vs.project.toml
  - .cursorrules
  - .claude/config.json
  - .kilocode/config.json
  - package.json (ai section)

Overlaps:
  - Feature flags in multiple places
  - Stack definition conflicts
  - Tool preferences differ
```

**Resolution Options:**
1. **consolidate** (recommended)
   - vs.project.toml as master
   - Other files add reference
   
2. **keep-separate**
   - May cause drift over time
   
3. **review-manual**
   - Manual review of each conflict

---

### CONF-004: GRACE Artifacts Conflict

**Description:** docs/*.xml from different sources/versions

**Detection:**
```
Files:
  - docs/development-plan.xml
  - docs/knowledge-graph.xml
  - docs/verification-plan.xml

Check:
  - VERSION attribute
  - Structure matches vibestart template
```

**Scenarios:**
- grace-marketplace format vs vibestart format
- Old vibestart version vs new
- Manual edits vs template structure

**Resolution Options:**
1. **migrate** (recommended)
   - Backup originals
   - Convert structure
   - Add missing files
   
2. **keep-add-missing**
   - No format changes
   - Create missing files only
   
3. **full-reset**
   - Replace all with templates
   - ⚠️ Loses existing content

---

### CONF-005: Session Log Conflict

**Description:** Existing session logs vs new structure

**Detection:**
```
Files:
  - docs/SESSION_LOG.md
  - docs/TASK_LOG.md

Check: Format matches vibestart template
```

**Resolution Options:**
1. **migrate-format**
   - Preserve content
   - Update structure
   
2. **keep-existing**
   
3. **archive-fresh**
   - Archive old logs
   - Start fresh

---

### CONF-006: Gitignore Conflict

**Description:** Missing entries for vibestart files

**Detection:**
```
File: .gitignore

Required entries:
  - docs/ai/private-rules.local.md
  - .env*.local
  - context_portal/
```

**Resolution:**
- Add missing entries (non-destructive)

## Global Defaults

### ~/.vibestart/global-defaults.toml

```toml
# vibestart Global Configuration
# Applies to ALL projects unless overridden

[version]
framework = "2.0.0"
installed = "2026-03-23T15:00:00Z"

[skills]
# Which skills are installed and from where
source = "vibestart"  # "vibestart" | "grace-marketplace" | "mixed"

[compatibility]
# How to handle conflicts
grace_source = "vibestart"  # "vibestart" | "grace-marketplace"
backup_originals = true
backup_path = "~/.vibestart/backups"

[defaults]
# Default values for new projects
features = [
  "grace",
  "session_log",
  "batch_mode"
]

[agent]
# Agent-specific settings
tool_prefixes = true
announce_actions = true
status_reports = true

[updates]
# Update behavior
auto_check = true
channel = "stable"  # "stable" | "beta" | "nightly"
```
