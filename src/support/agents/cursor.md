# Cursor Agent Support
<!-- Support: agents/cursor.md -->

## Installation

vibestart works with Cursor through skills located in `.cursor/skills/` directory.

## Skills Path

```
.cursor/skills/
├── vs-init/
└── grace/
    ├── grace-init/
    ├── grace-plan/
    ├── grace-status/
    ├── grace-refresh/
    └── ...
```

## Configuration

Cursor reads configuration from:
- `vs.project.toml` — project configuration
- `.cursorrules` — Cursor rules (can reference vibestart)

## Usage

### Initialize Project (includes AGENTS.md generation)
```
/vs-init
```

### GRACE Workflow
```
/grace-init
/grace-plan
/grace-verification
/grace-execute
/grace-status
/grace-refresh
```

## Custom Skills

To add custom skills, place them in `.cursor/skills/` with structure:
```
SKILL.md
assets/
  (optional templates)
```

## Rules File

Cursor uses `.cursorrules` file. To reference vibestart configuration:

```
# See vs.project.toml for AI tooling configuration
# Generated by vibestart

# Custom rules below
```

## Notes

- Skills are hot-reloaded with `/vs-init --resolve`
- Use `[SKILL:name]` prefix for all agent actions
- Keep `docs/SESSION_LOG.md` updated
