## Design-First Collaboration
<!-- Fragment: process/design-first.md -->

Plan before code. Follow requirements → Technology → Development Plan → Verification Plan → Code + Tests

## Flow

### 1. Requirements Analysis
Understand what you building and who will use it. Document decisions in `docs/requirements.xml`.

### 2. Technology Stack
Define stack in `docs/technology.xml`.

### 3. Development Plan
Design modules, data flows, and verification in `docs/development-plan.xml` and `docs/verification-plan.xml`.

### 4. Code + Tests
Implement according to contracts and verification criteria.

## Never Jump Straight to Code
When requirements, architecture, or verification intent are still unclear.
